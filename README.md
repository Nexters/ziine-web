# ziine-web

## 브랜치 전략

브랜치 이름은 이슈 번호와 유형을 포함하여 아래와 같이 작성합니다.

- (디폴트) `dev`
- `prod`: 배포할 때만 dev -> prod merge
- `feat/{#이슈 번호}`: 기능 추가 or 개선 작업
- `fix/{#이슈 번호}`: 버그 수정

## 커밋 메세지 작성 규칙

커밋 메세지는 다음과 같이 작성합니다.

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 기능 변경 없이 코드 개선
- `chore`: 기타 작업

## 이슈 및 PR 작성 규칙

템플릿에 맞춰 작성합니다.

## 코드 리뷰 규칙

각 리뷰 항목의 중요도는 `pn 룰`을 사용해 다음과 같이 구분합니다.

- **pn 1**: 꼭 반영해 주세요 (Request change)
- **pn 2**: 웬만하면 반영해 주세요. 다만 작업자 판단에 맡깁니다. (comment)
- **pn 3**: 그냥 사소한 의견입니다. (approve)
